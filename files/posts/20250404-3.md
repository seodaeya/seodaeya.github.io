## 🚀 코딩 에디터의 새로운 패러다임, Cursor

최근 개발 생태계에서 가장 뜨거운 관심을 받고 있는 <strong>Cursor(커서)</strong>는 단순한 플러그인 형태를 넘어 에디터 자체에 AI 인공지능을 내장한 <strong>VS Code 포크 기반의 AI 특화 코드 에디터</strong>입니다.

기존의 에디터들이 코드를 제안하는 데 그쳤다면, Cursor는 프로젝트 전체의 컨텍스트를 완벽히 이해하고 대규모 리팩토링과 멀티 파일 제어를 매끄럽게 수행합니다. VS Code의 설정을 단 한 번의 클릭으로 그대로 가져올 수 있어 마이그레이션 부담도 거의 없습니다.

---

## 🌟 Cursor의 핵심 기능 및 활용 방법

### 1. Composer (Cmd + I / Ctrl + I) - 멀티 파일 동시 제어
Cursor를 게임 체인저로 만든 기능입니다. 기존 챗 AI는 한 번에 하나의 파일만 수정 코드를 짜주었지만, Composer 모드는 전역 맥락을 파악하여 <strong>여러 파일의 코드를 동시에 수정 및 생성</strong>합니다.
* "이 API 엔드포인트를 추가하고, 프론트엔드 API 클라이언트 코드와 상태 관리 로직도 업데이트해줘"와 같은 요청을 한 번에 처리합니다.
* 수정한 내용이 Diff(차이점) 형태로 한 화면에 표현되어 개발자가 파일별로 검토한 후 승인(`Accept`)하거나 반려(`Reject`)할 수 있습니다.

### 2. Tab Autocomplete (탭 자동완성)
GitHub Copilot보다 훨씬 빠르고 영리하게 작동하는 자체 탭 자동완성 엔진을 사용합니다.
* 단순한 다음 단어 예측이 아니라, 개발자의 최근 편집 패턴을 파악하여 위아래 줄의 연관 코드를 알아서 고쳐주거나 주석을 기반으로 전체 메서드를 예측 제안합니다.

### 3. 코드 인라인 편집 (Cmd + K / Ctrl + K)
에디터 화면에서 코드 블록을 선택한 뒤 `Cmd + K`를 누르면 프롬프트 창이 뜹니다. 자연어로 수정 요구사항을 적으면 에디터 창 내부에서 즉시 코드를 변경해 줍니다. 

### 4. 맥락 기반 채팅 (Cmd + L / Ctrl + L)
에디터 우측에 탑재된 채팅 영역에서 소스 코드 전체를 대상으로 질문을 던질 수 있습니다.
* <strong>@Workspace:</strong> 프로젝트 내의 모든 파일을 인덱싱하여 프로젝트 구조에 맞는 최적의 대답을 찾습니다.
* <strong>@Files / @Web:</strong> 특정 파일이나 실시간 인터넷 검색 결과를 결합하여 보다 구체적인 개발 해결책을 얻을 수 있습니다.

## ⚙️ Cursor를 200% 활용하는 팁

### 1. `.cursorrules` / `.cursor/rules` 설정
프로젝트 루트 디렉터리에 `.cursorrules` 파일을 만들거나 `.cursor/rules/` 폴더 내에 규칙 마크다운 문서들을 작성해두면, Cursor AI가 답변을 생성할 때 해당 규칙을 강제 준수합니다.
```text
[규칙 예시]
- TypeScript 사용 시 반드시 Strict 옵션을 고려해 타이핑할 것.
- 컴포넌트는 Functional Component 형식을 지키며, 스타일은 TailwindCSS를 활용할 것.
- 주석은 반드시 한글로 작성할 것.
```
이 기능을 이용하면 팀의 코딩 컨벤션이나 특정 아키텍처 규칙을 AI에게 완벽하게 학습시킬 수 있습니다.

### 2. BYOM (Bring Your Own Model) 지원
Cursor는 기본 크레딧 요금제 외에도 개발자 본인의 <strong>OpenAI API Key</strong> 또는 <strong>Anthropic API Key</strong>를 직접 등록하여 사용할 수 있습니다. 이를 통해 사용량 제약 없이 저렴하고 유연하게 고성능 모델(GPT-4o, Claude 3.5 Sonnet 등)을 에디터에 직접 연결할 수 있습니다.
