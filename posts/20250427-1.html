🐘 PostgreSQL 개요

**PostgreSQL(포스트그레스큐엘)**은 엔터프라이즈급 성능과 신뢰성을 자랑하는 세계에서 가장 진보된 오픈소스 관계형 데이터베이스(RDBMS) 중 하나입니다. 강력한 복잡성 처리 능력, 뛰어난 표준 준수(SQL 표준), 확장 기능(JSON 데이터 처리, GIS 지리 정보 지원 등) 덕분에 스타트업부터 대기업까지 널리 사랑받고 있습니다.

본 가이드에서는 운영체제별 다운로드 방법부터 초기 설치, CLI 설정 및 필수 명령어까지 한눈에 쉽게 따라 하실 수 있도록 정리했습니다.

⬇️ 1단계: 설치 파일 다운로드

PostgreSQL은 운영체제별로 패키징된 공식 설치 마법사를 제공합니다.

  1. 공식 웹사이트 접속: PostgreSQL 다운로드 페이지로 이동합니다.
  2. 운영체제 선택: 본인의 OS(Windows, macOS, Linux)를 선택합니다.
  3. 인스톨러 페이지 이동: 화면 상단의 "Download the installer" 링크를 클릭하여 전문 배포사인 EDB에서 제공하는 설치 파일 리스트로 이동합니다.
  4. 버전 및 OS 선택: 일반적으로 가장 안정적이고 널리 사용되는 최신 LTS 버전을 추천합니다. 시스템 아키텍처(64비트 등)에 맞춰 인스톨러 다운로드를 진행합니다.

🛠️ 2단계: 단계별 설치 프로세스 (Windows 기준)

다운로드한 설치 파일(.exe)을 실행한 후 다음 단계를 차례대로 밟아 설치를 진행합니다.

1. 구성 요소(Components) 선택

설치 중간에 설치할 패키지를 묻는 창이 뜹니다. 특별히 제외해야 할 이유가 없다면 아래 4가지 기본값을 모두 체크하고 넘어갑니다.

  • PostgreSQL Server: 데이터베이스 핵심 엔진
  • pgAdmin 4: 데이터베이스 관리를 위한 공식 웹/앱 GUI 도구
  • Stack Builder: 드라이버나 모듈 등 추가 확장 프로그램 다운로더
  • Command Line Tools: psql 등 터미널 기반 유틸리티

2. 데이터 디렉터리(Data Directory) 설정

데이터베이스 실제 데이터 파일들이 저장될 기본 경로를 설정합니다. 디스크 여유 공간이 충분한 드라이브 경로로 지정하는 것이 좋습니다.

3. 데이터베이스 관리자 비밀번호 입력

기본 슈퍼유저인 postgres 계정의 비밀번호를 설정하는 화면입니다. 로컬 테스트 및 향후 연결을 위해 설정한 비밀번호는 분실하지 않도록 기록해 두세요.

4. 포트 번호(Port) 지정

PostgreSQL의 기본 포트 번호는 **5432**입니다. 이미 다른 서비스가 해당 포트를 사용 중이 아니라면 기본값 그대로 유지합니다.

5. 로케일(Locale) 선택

데이터 정렬 및 인코딩에 영향을 주는 로케일을 정합니다. 보통 [Default Locale] 또는 유니코드 지원을 위해 ko_KR.UTF-8 등을 설정합니다. 이후 다음 버튼을 눌러 설치를 완료합니다.

🚀 3단계: 초기 설정 및 연결 확인

GUI 환경 (pgAdmin 4) 활용

  1. 설치 완료 후 pgAdmin 4를 실행합니다.
  2. 최초 실행 시 마스터 비밀번호를 설정하라는 창이 뜹니다. 관리용 마스터 패스워드를 지정합니다.
  3. 좌측 사이드바의 Servers 영역을 펼친 뒤 설치된 서버(PostgreSQL X)를 더블 클릭합니다.
  4. 설치 과정에서 지정했던 postgres 관리자 비밀번호를 입력하면 데이터베이스 서버 연결이 활성화됩니다.
  5. 이제 마우스 우클릭으로 손쉽게 데이터베이스를 생성하고, 쿼리 툴(Query Tool)을 열어 SQL을 연습할 수 있습니다.

💻 4단계: Command Line (CLI) 환경 설정 및 기본 명령어

터미널이나 명령 프롬프트(CMD)에서 psql 명령어를 직접 사용하려면 환경 변수 설정이 필수적입니다.

환경 변수(Path) 등록 방법 (Windows)

  1. 시작 메뉴에서 '시스템 환경 변수 편집'을 검색하여 엽니다.
  2. 환경 변수 버튼을 클릭한 뒤, 시스템 변수 목록 중 Path를 찾아 편집을 클릭합니다.
  3. 새로 만들기를 누르고 PostgreSQL이 설치된 경로의 bin 폴더 경로를 추가합니다. (예: C:\Program Files\PostgreSQL\버전in)
  4. 확인을 누른 후 모든 명령 프롬프트 창을 새로 엽니다.

터미널 접속 및 기본 SQL 활용

새 터미널을 열고 다음 명령어를 입력하여 PostgreSQL 관리자 모드로 접속합니다.

psql -U postgres

(비밀번호를 요구하면 설치 시 등록했던 비밀번호를 입력합니다.)

📌 psql 내 유용한 CLI 명령어 목록

  • \l : 서버 내 전체 데이터베이스 목록 확인
  • \c [데이터베이스명] : 해당 데이터베이스로 전환(연결)
  • \dt : 현재 데이터베이스 내 전체 테이블 목록 확인
  • \q : psql 셸 종료

📌 최초 접속 시 권장 설정 SQL 쿼리 예시

-- 1. 새로운 전용 서비스 데이터베이스 생성
CREATE DATABASE my_app_db;

-- 2. 애플리케이션 전용 일반 사용자 계정 생성 (보안 권장)
CREATE USER my_app_user WITH ENCRYPTED PASSWORD 'safe_user_password';

-- 3. 새로 생성한 데이터베이스에 대해 일반 사용자에게 권한 부여
GRANT ALL PRIVILEGES ON DATABASE my_app_db TO my_app_user;

⚠️ 5단계: 흔히 겪는 설치 및 연결 문제 해결법

  • 포트 충돌 (Port 5432 is already in use): 다른 프로그램이 이미 5432 포트를 점유하고 있는 경우입니다. 설치 중 포트 번호를 5433 등으로 변경하여 설치하거나, 작업 관리자에서 기존에 돌고 있는 포트 관련 프로세스(예: Docker 컨테이너 등)를 종료해야 합니다.
  • 접속 인증 오류 (Password authentication failed): 비밀번호가 틀린 경우입니다. pgAdmin이나 psql 로그인 시 입력한 패스워드가 설치 단계에서 지정한 슈퍼유저 패스워드와 정확히 일치하는지 확인하세요.
  • 방화벽 및 로컬 접근 문제 (pg_hba.conf 설정): 외부 서버나 컨테이너에서 접근이 안 될 때는 PostgreSQL 설치 폴더 내 data/pg_hba.conf 파일 및 postgresql.conf 파일에서 외부 접근 허용 IP 목록(listen_addresses = '*')을 수정해 주어야 합니다.

📌 요약 및 체크리스트

설치를 완료한 후 아래 항목이 정상 동작하는지 확인해 보세요.

  • pgAdmin 4 정상 실행 및 관리자 서버 로그인 여부
  • 터미널에서 psql -U postgres 명령어로 콘솔 진입 가능 여부
  • CREATE DATABASE 쿼리가 성공적으로 동작하는지 여부
